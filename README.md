# Инструкция по работе с командами Git  

**Git** — самая популярная система контроля версий, но не единственная. Алгоритм работы подобных систем схож.

## Комманды необходимые для работы с локальным репозиторием:  
**git init** — инициализация локального репозитория;  
**git status** — получение информации о текущем состоянии репозитория;  
**git add** — добавление файлов к следующему коммиту;  
- **git add *filename*** — добавление указанного файла к следующему коммиту; 
- **git add .** — добавление всех файлов к следующему коммиту; 

**git commit -m "*message*"** — создание коммита;  
**git log** — вывод истории коммитов с их хеш-кодами.

>Команды можно вводить в терминале в Visual Studio Code, в коммандной строке.

## Комманды необходимые для работы с удаленным репозиторием:  
**git remote add origin *ссылка на ветку***   - инициализация удалённого ~~сетевого~~ репозитория  
**git branch -M main**  - создание ветки  
**git push -u origin main** - отправка данных в ветку удалённого репозитория  
**git config http.sslVerify false** - отключить проверку сертификатов для этого репозитория (если требуется)
